# Hermes Agent 功能模块梳理（代码级 + 重要等级 + 依赖关系）

> 说明：本仓库是 **Hermes Agent**（Nous Research）的源码。命令 `hermes claw migrate`（`hermes_cli/claw.py`）只是从 **OpenClaw** 项目迁移的入口，OpenClaw 本身的源码不在此仓库。下面所有模块均**对应真实文件**（已 grep 校验），重要等级依据「调用入度（被多少个 .py 文件引用）+ 在主链路上的位置」综合排序。

---

## 0. 重要等级定义

| 等级 | 含义 | 判定依据 |
|------|------|----------|
| **P0** | 核心骨架，删了系统无法运行 | 在主对话链路上 / 被全局依赖 |
| **P1** | 运行时控制，决定一次 turn 如何执行 | 高入度、conversation_loop 直接依赖 |
| **P2** | 能力模块，提供具体功能（工具/记忆/多Agent） | 可裁剪但定义产品能力 |
| **P3** | 接入层，对外入口 | 用户/平台进入系统的边界 |
| **P4** | 基础设施，支撑运行环境 | 环境隔离、调度、凭证、配置 |

## 1. P0 — 核心骨架（Core Skeleton）

| 模块 | 代码位置 | 职责 | 入度 |
|------|----------|------|------|
| **AIAgent** | `run_agent.py:333`（5,568 LOC） | Agent 主对象，持有会话状态、模型配置、`_cached_system_prompt`，方法 `chat()` (5325)、`run_conversation()` (5302 转发)、`interrupt()` (2376) | 高 |
| **对话循环 run_conversation** | `agent/conversation_loop.py:495`（模块级函数，非类，~4582 LOC） | 单次 turn 的主循环：组装上下文 → 调 LLM → 执行工具 → 收尾。整个系统的"心脏" | 10 |
| **工具注册表 ToolRegistry / ToolEntry** | `tools/registry.py:151` / `:77`（`__slots__` 类） | 所有工具的注册中心，`register()` 在 import 时自注册；`_generation` 计数失效缓存。`discover_builtin_tools()` 是模块级函数 (`:57`) | **117（最高）** |
| **会话持久化 SessionDB** | `hermes_state.py:667`（5,103 LOC） | SQLite 会话存储；`messages_fts`（contentless FTS5）+ `messages_fts_trigram`（CJK 子串检索）；sessions/messages 表 | 多 |
| **工具分发 model_tools** | `model_tools.py:901`（`handle_function_call`）/ `get_tool_definitions` (276) | LLM function-call → registry 查表 → 执行；构造发给模型的工具 schema | 中 |

**P0 内部依赖**：`conversation_loop` → `AIAgent`(状态) + `tool_executor` + `memory_manager` + `context_compressor`；`tool_executor` → `registry` + `model_tools`；几乎所有工具 → `registry`。

## 6. 核心依赖关系图（真实 import 边）

```
                       ┌───────────────────────────┐
   接入层 P3 ──────────▶│   AIAgent (run_agent.py)  │
 (cli/gateway/mcp/tui)  └─────────────┬─────────────┘
                                      │ run_conversation()
                                      ▼
              ┌───────────── agent/conversation_loop.py ─────────────┐
              │                       │                              │
              ▼                       ▼                              ▼
   ┌──────────────────┐   ┌────────────────────┐         ┌────────────────────┐
   │  P1 运行时控制    │   │   P2 工具系统       │         │   P2 记忆系统       │
   │ turn_context     │   │ tool_executor      │         │ memory_manager     │
   │ context_compress │   │   └▶ registry(117) │         │   └▶ memory_provider│

   │ error_classifier │   └─────────┬──────────┘         └─────────┬──────────┘
   └────────┬─────────┘             │                              │
            │                       ▼  delegate_task               │
            │            ┌────────────────────┐                    │
            │            │ P2 多Agent          │ 派生子 AIAgent ───┐ │

            │            │ async_delegation    │◀─────────────────┘ │
            │            └────────────────────┘                    │
            ▼                                                       ▼
   ┌─────────────────────── agent/auxiliary_client.py call_llm (37) ──────────┐
   │   anthropic / openai(chat_completion_helpers) / gemini / bedrock / codex  │
   └───────────────────────────────────┬──────────────────────────────────────┘
                                        ▼
                         P4 基础设施: environments / providers / cron / credentials / plugins
```

**关键依赖事实（已校验）**：
- `registry` 是被引用最多的模块（117 个文件），是工具系统的根基。
- `auxiliary_client.call_llm` 是唯一的模型调用出口（37 入度），所有适配器汇聚于此。
- `conversation_loop` 是运行时编排中心，向下依赖 P1+P2 全部子系统。
- `delegate_task` 反向依赖 `run_agent.AIAgent`（构造子 Agent），形成递归（受 `MAX_DEPTH=1` 限制）。
- 项目**不使用** LangGraph/LangChain（uv.lock 0 命中）；核心 LLM SDK 为 `openai==2.24.0`，`anthropic==0.87.0` 为可选 extra。
